# Como contribuir

Obrigado pelo interesse em melhorar o guia.

## Antes de enviar uma contribuição

- confirme o comportamento na versão atual do Vibium;
- prefira documentação e código oficial como fonte técnica;
- use exemplos públicos e reproduzíveis;
- não inclua dados pessoais, credenciais ou sistemas internos;
- escreva em português brasileiro claro;
- informe quando algo for uma interpretação ou experiência prática.

São especialmente bem-vindas contribuições verificadas sobre:

- JavaScript/TypeScript;
- Java;
- CLI do Vibium;
- Linux e macOS;
- testes reproduzíveis;
- correções causadas por novas versões.

O autor principal trabalha atualmente com Python e está iniciando os estudos de
MCP. Conteúdo de outras interfaces deve ser enviado por pessoas que consigam
testá-lo e mantê-lo.

## Padrão para documentar métodos

Inclua:

1. finalidade;
2. assinatura ou forma de uso;
3. exemplo mínimo;
4. resultado esperado;
5. retorno;
6. cuidados e limitações;
7. versão em que foi validado;
8. link para a fonte.

## Pull requests

Mantenha cada pull request focado em um assunto. Descreva o problema, a mudança
realizada e como o conteúdo foi verificado.
